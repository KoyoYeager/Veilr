# Veilr - アルゴリズム仕様書

## 1. シートモード（乗算ブレンド）

### 1.1 原理

実物の赤シートと同じ**カラーフィルター**を再現する。単純な半透明重ね合わせ（αブレンド）ではなく、**乗算ブレンド（Multiply）**を使用する。

### 1.2 計算式

各ピクセルのRGBチャネルに対して：

```
結果.R = ピクセル.R × シート色.R / 255
結果.G = ピクセル.G × シート色.G / 255
結果.B = ピクセル.B × シート色.B / 255
```

### 1.3 効果（シート色 = 赤 (255, 0, 0) の場合）

| 元のピクセル | 計算結果 | 視覚効果 |
|---|---|---|
| 白 (255, 255, 255) | (255, 0, 0) | 赤く見える |
| 赤文字 (255, 0, 0) | (255, 0, 0) | 背景と同色 → **文字が消える** |
| 黒文字 (0, 0, 0) | (0, 0, 0) | 黒のまま → **読める** |

### 1.4 キャプチャ方式

リアルタイムキャプチャではなく、以下のタイミングでワンショットキャプチャ：

- ウィンドウ初回表示時
- ドラッグ移動完了時（`DragMove()` 復帰後）
- リサイズ完了時（300msデバウンス後）
- モード切替時
- F5 / スペースキーで手動リフレッシュ

点滅を防ぐため、タイマーループは使用しない。

## 2. 消去モード（CIE Lab色差による色除去）

### 2.1 色空間の選択

**HSV色空間の問題点：**
- 黒 (0,0,0) と赤 (255,0,0) の色相Hがどちらも 0
- 閾値を広げると黒文字が赤として誤検出される
- 知覚的に均一でない

**CIE Lab色空間の利点：**
- 人間の色知覚に沿った均一な色空間
- 黒 (L=0, a=0, b=0) と赤 (L=53, a=80, b=67) のΔE ≈ 117 → 完全に分離
- 明度(L)と色味(a,b)が独立 → 明暗に関係なく「赤」を特定可能

### 2.2 RGB → CIE Lab 変換

```
1. sRGBガンマ補正（線形化）
   linear = v > 0.04045 ? ((v + 0.055) / 1.055)^2.4 : v / 12.92

2. 線形RGB → XYZ（D65白色点、sRGBマトリクス）
   X = 0.4124564·R + 0.3575761·G + 0.1804375·B
   Y = 0.2126729·R + 0.7151522·G + 0.0721750·B
   Z = 0.0193339·R + 0.1191920·G + 0.9503041·B

3. XYZ → Lab
   L = 116·f(Y/Yn) - 16
   a = 500·(f(X/Xn) - f(Y/Yn))
   b = 200·(f(Y/Yn) - f(Z/Zn))
   f(t) = t^(1/3)  (t > 0.008856), else 7.787t + 16/116
```

### 2.3 3層マッチングアルゴリズム

#### 第1層：直接マッチ（Lab色相距離）

```
色相距離 = √((a_pixel - a_target)² + (b_pixel - b_target)²)
```

明度(L)を無視し、色味の距離のみで判定。これにより：
- 暗い赤も明るい赤も同じ「赤」として検出
- 黒/白/灰色は色味成分(a,b)が原点付近 → 遠いので検出されない

**判定条件：** 色相距離 ≤ 閾値 (`tolerance × 0.45 × 1.35`)

#### 第2層：角度マッチ（アンチエイリアス対応）

アンチエイリアスされたテキストの辺は、ターゲット色と白の中間色になる。色相距離は大きくなるが、Lab a-b平面上の**色相角度**は保持される。

```
ターゲット角度 = atan2(b_target, a_target)
ピクセル角度 = atan2(b_pixel, a_pixel)
角度差 = |ターゲット角度 - ピクセル角度|
```

**判定条件：** 角度差 ≤ 閾値×1.2° かつ ピクセル彩度 > 3

| ピクセル種類 | 色相距離 | 色相角度差 | 第1層 | 第2層 |
|---|---|---|---|---|
| 赤テキスト本体 | 8 | 0-5° | ✅ | ✅ |
| アンチエイリアス辺 | 31 | 12-23° | ❌ | ✅ |
| 茶色ヘッダー | 28 | 28° | ❌ | ❌ |
| 黒テキスト | 54 | 47° | ❌ | ❌ |

#### 第3層：条件付き膨張（最も薄いエッジ用）

2回のパスで、マーク済みピクセルに3方向以上隣接する未マークピクセルを追加マーク。

**条件：**
- 4連結の隣接マーク済みピクセルが3個以上
- ピクセルのLab彩度 > 1.5（完全無彩色を除外）
- 色相角度差 ≤ 25°

### 2.4 置換アルゴリズム

マーク済みピクセルは、最寄りの非マークピクセルの色で置換される。

```
半径1から50まで拡大する矩形の外周を走査：
  非マークピクセルが見つかれば → そのRGBで置換
  見つからなければ → 半径を+1して継続
フォールバック: 白 (255, 255, 255)
```

### 2.5 許容範囲スライダー（UI）

ユーザーの「許容範囲」設定（0-100）から閾値へのマッピング：

```
H閾値 = tolerance × 0.45    （0-45）
S閾値 = tolerance × 1.0     （0-100）
V閾値 = tolerance × 1.0     （0-100）

色相距離閾値 = H閾値 × 1.35  （0-60.75）
角度閾値 = 色相距離閾値 × 1.2°
```

## 3. カラーピッカー（スポイト）

### 3.1 実装方式

画面全体をスナップショットとして一括キャプチャし、不透明なフルスクリーンウィンドウに表示。

**この方式の利点：**
- クリックが確実に動作（透明オーバーレイのhit test問題を回避）
- マルチモニター対応（VirtualScreen全域をカバー）
- カーソル位置と表示位置の一致（DPI変換を一括処理）

### 3.2 ズームプレビュー

- カーソル周辺 21×21 ピクセルを切り出し
- `BitmapScalingMode.NearestNeighbor` で8倍表示
- 中心ピクセルにコントラスト色の十字マーカーを描画
- プレビューウィンドウは `IsHitTestVisible=false` でクリック透過

### 3.3 座標系

| 処理 | 座標系 |
|---|---|
| スナップショット取得 | 物理ピクセル（WinForms Screen.Bounds） |
| クリック位置 → 色取得 | 物理ピクセル（GetCursorPos → スナップショットインデックス） |
| プレビュー配置 | 物理ピクセル ÷ DPIスケール → WPF DIP |
| オーバーレイウィンドウ配置 | 物理ピクセル ÷ DPIスケール → WPF DIP |

## 4. 文字色自動選定

シートウィンドウのバー・ボタンの文字色は、シート色の輝度に応じて自動選定。

```
輝度 = 0.299 × R + 0.587 × G + 0.114 × B
文字色 = 輝度 > 140 ? 黒 : 白
```
